About This Project

GRC Compliance Dashboard — Built by Oussama Khoulaidi

⚠ FICTIONAL DATA DISCLAIMER

All company data, findings, risks, vendors, and compliance figures in this dashboard are entirely fictional and were created for portfolio demonstration purposes only. DataFlow Qatar does not exist. No real organisation, person, or event is referenced. The data simulates a realistic Q1 2026 internal GRC audit scenario for a fictional Qatari fintech company.

Project Overview

This is a live, data-driven GRC (Governance, Risk & Compliance) dashboard built as a portfolio project to demonstrate practical expertise in ISO 27001:2022, Qatar PDPPL, PCI-DSS v4.0, and enterprise risk management.

The dashboard reads directly from Microsoft Excel (.xlsx) audit workbooks hosted on GitHub Pages. When any Excel file is updated and pushed to GitHub, the dashboard automatically reflects the new data — no code changes required. This mirrors how the tool would work in a real organisation: an auditor fills in the Excel templates with real audit findings, and the dashboard becomes a live compliance view.

Fictional Company: DataFlow Qatar

Industry: Fintech / Digital Payments & Card Processing

Location: West Bay Financial District, Doha, Qatar

Size: ~280 employees, founded 2018

Business: DataFlow Qatar provides digital payment processing, card issuing, and financial data analytics services to retail and enterprise clients across the GCC. It processes an average of 4.2M transactions per month and is regulated by the Qatar Central Bank (QCB).

Audit scenario: Q1 2026 internal audit conducted in preparation for ISO 27001:2022 certification (target: Q4 2026). The audit also assessed compliance with Qatar PDPPL (Law No. 13 of 2016) and PCI-DSS v4.0.

How It Works

The dashboard is a single index.html file with zero backend dependencies. Here's the architecture:

📊
SheetJS
Reads .xlsx files directly in the browser
📈
Chart.js v4
Renders charts from parsed data
🌐
GitHub Pages
Static hosting, no server needed
📝
Excel (xlsx)
Single source of truth for all data

The page loads 5 Excel files in parallel on startup: ISO27001_Risk_Register.xlsx, ISO27001_Statement_of_Applicability.xlsx, Vendor_Risk_Assessment.xlsx, Qatar_PDPPL_Compliance_Checklist.xlsx, and Remediation_Tracker.xlsx. All calculations and charts are computed dynamically from the parsed data.

Using With Real Audit Data

To adapt this dashboard for a real organisation:

  • Download the Excel templates from the GitHub repository
  • Fill in the sheets with real audit findings, risks, vendor assessments, and compliance data — following the exact column structure
  • Push the updated Excel files to GitHub (the HTML file requires no changes)
  • The dashboard will automatically update within ~60 seconds (GitHub Pages cache)

Important: use hardcoded values (not Excel formulas) in data cells, as SheetJS reads computed cell values from the spreadsheet.
